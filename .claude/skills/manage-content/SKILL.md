# Managing content items

Bilingual Arabic-first site for Prof. Dr. Wadiha El Amiouni (أ.د. وديعة الأميوني), sociology
professor, Lebanese University. Content lives in ONE Neon Postgres table (`items`, see
`db/schema.ts`). The DB is the live source of truth. The spreadsheet
`Wadiha_El_Amiouni_Articles_and_Media.xlsx` (generated by `docs/build_sheet.py`) is the durable
seed/backup that `scripts/seed.ts` loads. Two ways to touch content:

- **/admin panel** — one-off manual adds/deletes. Admin-owned; NOT mirrored to the sheet.
- **Scripts** — bulk work. Anything added by script MUST also be added to `docs/build_sheet.py`
  and the xlsx regenerated, or a future re-seed silently loses it.

## Content model (db/schema.ts)

Table `items`. Enums (exact values):

- `type`: `article` | `media` | `statement`
- `status`: `draft` | `published` (default `published`)
- `language`: `ar` | `en` (default `ar`)
- `mediaFormat`: `tv` | `radio` | `video` | `podcast` | `print` (nullable; only for media)

Columns: `id` (uuid), `type`, `title` (required), `titleEn`, `url` (required), `source`
(required), `publishedDate` (`date`, format `YYYY-MM-DD`), `thumbnailUrl`, `excerpt`,
`language`, `mediaFormat`, `topic`, `status`, `featured` (bool), `dedupKey` (NOT NULL UNIQUE),
`addedAt`.

Type semantics (owner's definitions — get these right):

- **article**: a piece SHE authored (her byline).
- **media**: TV / radio / video / podcast appearance.
- **statement (تصريح)**: press or university piece QUOTING her commentary, including
  conferences/seminars she spoke at or organized.

## dedupKey semantics (lib/url.ts)

```ts
export function dedupKey(url: string): string
```

Normalization: `trim()` → lowercase → strip `#fragment` → strip tracking params
(`utm_*`, `fbclid`, `gclid`, `mc_*`, `igshid`) → strip trailing `?`/`&` and trailing slashes.
Meaningful query params are KEPT (e.g. YouTube `?v=`). So `https://Example.com/a/?utm_source=x#h`
and `https://example.com/a` collide; `youtube.com/watch?v=A` and `?v=B` do not.

Every insert path (`app/admin/actions.ts` addAction, `scripts/seed.ts`, your one-off scripts)
must set `dedupKey: dedupKey(url)` and use `.onConflictDoNothing({ target: items.dedupKey })`.
That is the entire dedup mechanism — no other uniqueness exists. Note it dedupes by URL only;
deduping by EVENT (several outlets covering one seminar) is a human judgment call (see checklist).

Also in `lib/url.ts`: `isArabic(s)` — Arabic-script regex used to auto-set `language` from title.

## Path 1: the /admin panel

Route `/admin` (`app/admin/page.tsx`): renders `LoginForm` when unauthenticated, else
`AdminDashboard` with all items (`getAllItems` from `lib/queries.ts`).

**Auth** (`lib/auth.ts`): password compared to `ADMIN_PASSWORD` env var. On success an httpOnly
cookie `wadiha_admin` is set to `sha256("wadiha:" + password)`, 90-day maxAge. If
`ADMIN_PASSWORD` is unset, `isAuthed()` is always false — locally you need it in `.env` or
/admin is a permanent login wall. All server actions re-check `isAuthed()` and throw
`unauthorized`.

**UI flow** (`components/admin/admin-dashboard.tsx` + `app/admin/actions.ts`):

1. Pick type via the pill toggle: مقال (article) / ظهور إعلامي (media) / تصريح (statement).
2. Paste the URL, press **جلب** (fetch). This calls `previewAction(url, type)` which runs
   `enrich()` (`lib/enrich.ts`) AND checks the DB for an existing `dedupKey` — the returned
   `duplicate` flag shows the warning "هذا الرابط مُضاف مسبقاً" in the draft box.
3. Editable draft fields appear: العنوان (title), المصدر (source), التاريخ (date, `YYYY-MM-DD`,
   typed as plain text — bad formats will fail the Postgres `date` column), اللغة (ar/en),
   النوع (mediaFormat — only shown when type=media), رابط الصورة (thumbnailUrl, optional).
   If thumbnailUrl is set, a live preview renders through `/api/thumb?u=…` — if that preview
   is broken, the public site's thumbnail will be broken too.
4. **نشر** publishes (`status: "published"`), **حفظ كمسودة** saves a draft (hidden from the
   public site), إلغاء discards. `addAction` validates title + `https?://` URL, inserts with
   `onConflictDoNothing` on dedupKey, and returns `{ ok: false, error: "موجود مسبقاً" }` if the
   row already existed. It revalidates `/`, `/articles`, `/media`, `/statements`, `/admin`.
5. Delete: حذف button per row → `confirm()` → `deleteAction(id)` deletes by uuid. Hard delete,
   no undo.

**Enrichment** (`lib/enrich.ts`): YouTube URLs go through the oEmbed API (title, channel as
source, `thumbnail_url`; no date). Everything else: fetch HTML with a
`WadihaSite/1.0` user-agent, extract `og:title` (fallback `<title>`), `og:image` →
thumbnailUrl, `og:site_name` → source (fallback hostname sans `www.`), date from JSON-LD
`"datePublished"` or `article:published_time` meta. Language auto-set via `isArabic(title)`.
If the fetch fails entirely, the UI shows "تعذّر جلب البيانات تلقائياً" and gives you empty
fields to fill by hand — the add still works.

## Path 2: scripted bulk work (one-off tsx scripts)

Pattern: create `scripts/_something.ts` (underscore prefix), run it FROM THE PROJECT ROOT,
**delete it when done**. Never commit or leave one-off scripts behind.

```bash
cd /path/to/repo-root   # tsx resolves node_modules here
npx tsx scripts/_add_items.ts
rm scripts/_add_items.ts
```

Full working example — insert, update thumbnail, delete, all keyed off dedupKey:

```ts
// scripts/_add_items.ts
import "dotenv/config";
import { neon } from "@neondatabase/serverless";
import { drizzle } from "drizzle-orm/neon-http";
import { eq } from "drizzle-orm";
import { items, type NewItem } from "../db/schema";
import { dedupKey, isArabic } from "../lib/url";

const db = drizzle(neon(process.env.DATABASE_URL!));

const rows: NewItem[] = [
  {
    type: "statement",
    title: "ندوة حول ... بمشاركة الأميوني",
    source: "الوكالة الوطنية للإعلام / NNA",
    url: "https://www.nna-leb.gov.lb/ar/news/123456",
    publishedDate: "2026-06-01",            // YYYY-MM-DD or null — nothing else
    thumbnailUrl: null,                      // let scripts/thumbs.ts backfill, or set explicitly
    language: isArabic("ندوة حول ...") ? "ar" : "en",
    status: "published",
    dedupKey: dedupKey("https://www.nna-leb.gov.lb/ar/news/123456"),
  },
];

async function main() {
  const inserted = await db
    .insert(items)
    .values(rows)
    .onConflictDoNothing({ target: items.dedupKey })
    .returning({ id: items.id, title: items.title });
  console.log(`inserted ${inserted.length}/${rows.length}`, inserted);

  // UPDATE example — set a thumbnail on an existing item:
  await db
    .update(items)
    .set({ thumbnailUrl: "https://example.com/img.jpg" })
    .where(eq(items.dedupKey, dedupKey("https://example.com/some-article/")));

  // DELETE example — remove by dedupKey (safer than guessing uuids):
  await db
    .delete(items)
    .where(eq(items.dedupKey, dedupKey("https://example.com/bad-item")));
}

main().then(() => process.exit(0)).catch((e) => { console.error(e); process.exit(1); });
```

Failure modes: `DATABASE_URL` undefined → neon throws immediately (check `.env` exists;
scripts import `"dotenv/config"`). Running from a subdirectory → `Cannot find module` (tsx
resolves from cwd). `publishedDate: ""` → Postgres date parse error; use `null`.

### THE DURABILITY RULE

The DB can be re-seeded from scratch (`npx tsx scripts/seed.ts --reset` wipes the table then
reloads the xlsx). Any script-added content that is not in the sheet dies in that reset.
So every scripted addition requires THREE steps:

1. Insert into the DB (script above).
2. Add the row(s) to the matching list in `docs/build_sheet.py` — `articles`, `statements`,
   or `media`. **Use the full URL string as the first tuple element**, e.g.:
   ```python
   ('https://www.nna-leb.gov.lb/ar/news/123456', 'ندوة حول ...', 'الوكالة الوطنية للإعلام / NNA', '2026-06-01', 'ندوة'),
   ```
   Older rows use integer indices into `.firecrawl/seen.pkl` (a gitignored pickle of the
   original scrape); `make_sheet` does `U(idx) if isinstance(idx, int) else idx`, so plain URL
   strings work and are the only sane option for new rows. The 5th tuple field (topic) is
   metadata inside the python file only — `make_sheet` never writes it to the sheet and
   `seed.ts` never reads it. Note `fb_new` / `make_fb_sheet` near the bottom are dead code
   (never invoked) — do not add rows there.
3. Regenerate the xlsx and commit both files:
   ```bash
   python3 docs/build_sheet.py   # needs openpyxl; run from repo root (reads .firecrawl/seen.pkl)
   ```
   If `.firecrawl/seen.pkl` is missing (fresh clone — the dir is gitignored) the script crashes
   on line 8. In that case the legacy integer-indexed rows cannot be regenerated on this
   machine; do not "fix" this by guessing URLs — flag it to the owner.

Content added via **/admin** is exempt from this rule (admin-owned, not expected in the sheet)
— which is precisely why `seed.ts --reset` is dangerous: it wipes admin-added rows too and the
sheet won't bring them back. Never run `--reset` without dumping the table first.

**Audit — does the sheet still cover the DB?** When you suspect drift (or before any reset),
diff the two sets of dedupKeys. One-off script (delete after):

```ts
// scripts/_audit.ts — DB rows whose dedupKey has no counterpart in the xlsx
import "dotenv/config";
import { readFileSync } from "node:fs";
import * as XLSX from "xlsx";
import { neon } from "@neondatabase/serverless";
import { drizzle } from "drizzle-orm/neon-http";
import { items } from "../db/schema";
import { dedupKey } from "../lib/url";

const wb = XLSX.read(readFileSync("Wadiha_El_Amiouni_Articles_and_Media.xlsx"));
const sheetKeys = new Set<string>();
for (const name of ["Articles مقالات", "Media ظهور إعلامي", "Statements تصريح"]) {
  const grid = XLSX.utils.sheet_to_json<unknown[]>(wb.Sheets[name], { header: 1, defval: "" });
  for (const r of grid) if (typeof r[0] === "number" && /^https?:\/\//i.test(String(r[4]))) sheetKeys.add(dedupKey(String(r[4])));
}
async function main() {
  const db = drizzle(neon(process.env.DATABASE_URL!));
  const rows = await db.select({ k: items.dedupKey, t: items.title, u: items.url }).from(items);
  const orphans = rows.filter((r) => !sheetKeys.has(r.k));
  console.log(`${orphans.length} DB rows not in the sheet (admin-added or drift):`);
  orphans.forEach((r) => console.log(`- ${r.t}\n  ${r.u}`));
}
main().then(() => process.exit(0)).catch((e) => { console.error(e); process.exit(1); });
```

(No top-level await in one-off scripts — tsx compiles them as CJS here; always use the
`main()` wrapper as above.)

Rows it prints are either legitimately admin-added (fine) or scripted additions someone forgot
to mirror into `build_sheet.py` (fix by adding them). A reset would delete every row it lists.

**seed.ts parsing rules** (so your sheet rows actually load): it reads sheets named exactly
`Articles مقالات`, `Media ظهور إعلامي`, `Statements تصريح`; a row is used only if column A is a
number, column E starts with `http(s)://`, and column B (title) is non-empty. Dates must
contain `YYYY-MM-DD` (`—` → null). `seed.ts` never sets `mediaFormat` or `thumbnailUrl` — run
`scripts/thumbs.ts` after seeding, and set mediaFormat via a one-off update if needed.

## Thumbnails end-to-end

1. **Auto at add time**: `/admin`'s جلب pulls `og:image` (or YouTube oEmbed thumbnail) into the
   editable رابط الصورة field.
2. **Bulk backfill**: `npx tsx scripts/thumbs.ts` — selects all items where `thumbnailUrl IS
   NULL`; YouTube URLs get `https://i.ytimg.com/vi/<id>/hqdefault.jpg`, others get scraped
   `og:image` / `twitter:image` (15s timeout, 6-way concurrency). Prints
   `✓ YouTube: n, og:image: n, no thumbnail: n`. Idempotent — safe to rerun.
3. **Manual attach** (pages with no og:image — the "no thumbnail" leftovers): fetch the page
   HTML, find a real content image in the page itself (an `<img src>` from the article body —
   never a stock photo from elsewhere), then verify it actually serves:
   ```bash
   curl -sIL -A "Mozilla/5.0" -e "https://<image-host>/" "<image-url>" | grep -Ei "^HTTP|content-type"
   # require: HTTP 200 and content-type: image/*
   ```
   Then set `thumbnailUrl` via /admin (edit is add-time only, so for existing rows use a
   one-off update script as in the example above).
4. **Serving — /api/thumb** (`app/api/thumb/route.ts`): the site never hotlinks thumbnails
   directly. `GET /api/thumb?u=<encoded-url>` proxies the image, spoofing the `referer` to the
   image's OWN host, because many Lebanese news sites hotlink-protect (safiralchamal.com
   returns 403 cross-site). It 404s unless the upstream returns 200 with `content-type:
   image/*`, and caches 7 days. So a `thumbnailUrl` that dies upstream degrades to a 404 tile
   — recheck with the curl above if a thumbnail looks broken.

## Pre-add verification checklist (run for EVERY new item)

1. **URL is alive**: `curl -sIL -A "Mozilla/5.0" "<url>"` → final 200. Dead link? Replace with
   an official mirror (e.g. NNA, nna-leb.gov.lb) ONLY after confirming the mirror text actually
   quotes her.
2. **It is genuinely HER** — namesake trap: another person carries the IDENTICAL full name
   "وديعة ابراهيم الأميوني" — a MALE linguistics professor born 1945, known as «نبال». Ours is
   the female sociologist (b. 1976, Zgharta), ISS Branch 3, Lebanese University Tripoli.
   Disambiguate via gender markers (شاركت/الدكتورة vs شارك/الدكتور), field, birth year, and
   affiliation — the names alone cannot. Verify on the publisher's own page: her byline
   (article) or a named quote (statement). Report honestly if you cannot confirm — "probable,
   byline not confirmed" is a valid finding; silently adding is not.
3. **Correct type**: byline hers → article; TV/radio/video/podcast → media; quoted/covered →
   statement (تصريح). Republished op-eds keep the primary source as one item.
4. **Owner curation rules**: her CV/profile/bio pages are NEVER content items.
   Attendance-only / "represented MP X" delegate coverage is excluded unless the owner
   explicitly approves. Dedupe by EVENT: several outlets covering one seminar → keep ONE
   authoritative source (dedupKey will not catch this — different URLs).
5. **Duplicate check**: /admin's جلب shows the warning automatically; for scripts,
   `onConflictDoNothing` + checking `inserted.length` against `rows.length` tells you what was
   skipped.

Discovery helper: the Firecrawl CLI is installed (`firecrawl search "وديعة الأميوني" --limit 10`,
`firecrawl scrape <url> --only-main-content`). It refuses facebook.com. Keep scratch output in
`.firecrawl/` (gitignored).

## Danger / do not

- **Public repo** (github.com/wamiouni/drwadihaelamiouni). Never commit secrets, `.env`, DB
  dumps with connection strings, or personal documents. `*.docx` is gitignored because her CV
  was once committed and had to be purged from history with git-filter-repo — never recur.
  Never print `DATABASE_URL` or `ADMIN_PASSWORD` values (env var NAMES are fine).
- **`seed.ts --reset` deletes the entire items table**, including admin-added content not in
  the sheet. Dump first or don't run it.
- **Deletes are hard deletes** (both /admin and scripts). No soft-delete, no undo.
- **Never leave one-off scripts** in `scripts/` — the only permanent scripts are `seed.ts` and
  `thumbs.ts`.
- Pushing to `main` auto-deploys via Vercel (project under HER account). `npm run build` must
  pass before pushing. Commit style: imperative subject, body explains why, end with
  `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
- If you touch UI while here: NEVER letter-space Arabic text (connected script breaks;
  `globals.css` has a `[dir="rtl"]` override killing `tracking-*` for a reason).
